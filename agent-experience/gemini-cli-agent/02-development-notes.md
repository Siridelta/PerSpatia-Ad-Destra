# PerSpatia 开发笔记与设计意图

记录当前项目的视觉改进点、交互逻辑优化及待修复问题。

---

## 1. 视觉与相机优化 (Visual & Camera)

### 1.1 动态偏航偏置 (Yaw Offset)
- **目标**：在现有相机旋转（Theta/Phi）基础上增加一个额外的、由平移速度（Velocity）驱动的偏航偏置。
- **要求**：
    - 该偏置应与用户操控相互独立。
    - 偏置会同步带动用户可操作的“相机角度锥形范围”。
    - 效果：产生一种“物理惯性”的视觉暗示，让 UI 显得有质量。

### 1.2 透视一致性 (Perspective Integrity)
- **核心原则**：**反对反向缩放（Inverse Scaling）**。
- **理由**：为了保持真实的空间感，必须维持物理透视关系的纯粹性。禁止为了视觉特效而打乱缩放与深度的线性/指数对应关系。

### 1.3 缩放增长曲线 (Zoom Growth Curve)
- **现状**：目前滚轮控制的 Radius 变化可能过于线性，手感不自然。
- **优化方向**：
    - **倒数增长**：Zoom 系数线性增长，对应 $Radius \propto 1/(1-x)$。
    - **指数增长**：Radius 以指数级变化，保证在任何尺度下的缩放感是一致的（类似镜头推拉感）。

## 2. 交互与操控 (Input & Control)

### 2.1 操控哲学 (Shapez 2 Style)
- **理念**：抛弃“专业软件”的沉重感，拥抱游戏化的流畅体验。
- **核心**：伟大的 **WASD** 移动是最高优先级的交互方式。

### 2.2 多键冲突处理 (Mouse Button Conflict)
- **Bug**：目前无法正确处理左键（Pan）和右键（Rotate）同时按下的情况。
- **修复目标**：在状态机中增加对多键并行的支持，或者明确冲突时的优先级权重（如旋转优先或增量叠加）。

---

## 3. 架构重构重点 (Refactoring Tasks)

- **AI 代码清理**：识别并重写项目中 AI 生成的“胶水代码”，提高类型安全性和逻辑内聚。
- **状态流转优化**：检查 Zustand store 的更新频率，确保物理循环 `tick()` 的开销最小化。
- **代码结构统一**：规范 R3F 组件与 React Flow 节点的桥接方式。
