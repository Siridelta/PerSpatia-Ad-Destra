# PerSpatia 开发笔记与设计意图

本笔记记录了项目在交互、视觉和架构上的演进逻辑，用于辅助 AI Agent 快速理解非直观的设计决策。

---

## 1. 交互哲学：游戏化体验 (Game-like UX Architecture)

### 1.1 设计动机
PerSpatia 致力于打破“传统生产力工具”与“现代 3D 游戏”之间的壁垒。我们认为，顺滑、直觉且具有物理反馈的操控能够显著提升长感官工作下的愉悦度。

### 1.2 核心交互模式
- **WASD 优先**：平移操作被视为最高优先级的交互方式之一。它不仅是 2D 画布的平移，更是相机在 3D 墙面上的“扫掠”。
- **轨道旋转 (Orbit)**：通过右键驱动，提供具有阻尼感的视角倾斜，增强空间深度感。
- **物理反馈**：所有的交互都应带有物理模拟（阻尼、惯性、碰撞投影），消除生硬的数位感。

## 2. 相机物理系统 (Camera Physics)

### 2.1 动态漂移 (Dynamic Drift / Yaw)
- **机制**：由平移速度（Velocity）线性感应产生的微小角度偏航。
- **目的**：提供视觉上的“质量感（Mass）”和“惯性暗示”，让用户感知到视角移动的快慢。
- **实现**：`drift = velocity / radius`，确保在不同缩放层级下偏航的视角位移在像素层面体感一致。

### 2.2 锥体约束与滑动 (Conical Clamp & Sliding)
- **约束**：手动旋转被限制在 20 度锥体内，防止用户在无限 3D 空间中“迷失”。
- **滑动物理**：当视角撞击 20 度边界时，惯性速度会被投影到边界切线上，实现“贴边滑行”效果，而非生硬撞死。

### 2.3 指数缩放 (Exponential Zoom)
- **机制**：Radius 的变化运行在对数空间（Logarithmic Space）。
- **理由**：保证在任何缩放尺度下（极近或极远），同等强度的滚轮操作产生的“视觉缩放比例”是恒定的。

## 3. 架构准则

### 3.1 唯一事实源 (SSOT)
相机状态由底层 Zustand Store 绝对控制。上层渲染层（R3F / React Flow）仅作为 Store 状态的视图投影。严禁任何从渲染层向物理层的“反向数据流”。

### 3.2 增量驱动 (Delta-based)
所有的位置更新应基于“位移增量”而非“绝对目标”。这确保了在存在 Clamp 等非线性约束的情况下，控制系统依然能够实时响应反向输入，避免饱和死区。
