# PerSpatia 视觉与空间感（设计意图笔记）

本文说明 **r3f-reactflow-nextui** 里「2D 画布 + 3D 景深」这一美学**想达成什么**，与具体相机公式无关；实现细节见 [camera-architecture.md](./camera-architecture.md)。

---

## 1. 我们为什么要做 3D 化

目标**不是**让用户像建筑漫游一样 **360° 绕场景飞一圈**。核心目的是 **空间感与纵深**：让用户在编辑信息流、节点图时，仍能联想到「屏幕背后」还有层次、机制与数据在运转——把抽象的系统 **在想象里** 变得可触摸一点。

参考气质上接近 **DATA WING 游戏**：在这个游戏里，叙事里玩家（一只 DATA WING）和 Mather（系统里的一个内置 AI，控制系统的大部分，并指挥调遣所有 DATA WING）是存在于"用户"的一台手机中的，游玩时玩家作为一只在地图里飞行的 DATA WING，长期看到的是 **「平面内容 + 背后深邃的 3D 空间」**——那种深度会自然引导玩家去脑补：深处有什么规则、信号、代码在动。PerSpatia 希望借类似的 **plane-on-3D** 语言，唤起我们对「信息世界」的某种原初想象——它只在上个时代自然地存在过，而在今天已经被潮流忽视。它是我们人类初次步入信息时代时，最 intuitive、最自然、最朴素的想象、也是我们在目前跑偏的潮流声量下被长期忽视的、然而却是我们心底里原本最想要的那种，视觉感受。PerSpatia 和 RcA 想要做的是让我们找回这种感觉，找回这种目前只被称为 FUI、然后被人们高高挂起、被贴上不切实际标签的、那种感觉、那种风格。PerSpatia 并不是单纯多一个旋转玩具，或者炫技。

项目组在 **assets**（或等价资源目录）中保留有 DATA WING 的录屏，曾作为把上一代纯 2D 子工程（如 `solution-react-flow`）推进到「类 DATA WING 的 3D 世界」时的**视觉参考**（启动 r3f 子工程迭代时曾把录屏交给模型并直接下指令）。若录屏未进 Git，以本地/团队约定路径为准。

## 3. 空间约定与相机预期（与 camera-architecture 对齐）

**世界系里「板子」不是地板，是一面墙：**

- **XY 平面**：**竖直**的，相当于 **一面墙**；React Flow 画布贴在这面墙上（实现里仍是 **Z = 0** 平面）。
- **Y 轴**：**竖直**（沿墙上下）。
- **X 轴**：沿墙水平方向（例如向右）。
- **Z 轴**：**水平纵深**——垂直于这面墙、向场景里「伸进去 / 拉出来」的方向；相机大致在 **Z 正方向一侧** 看向墙上的注视点，和「屏幕外往里看」的纵深一致。

**「垂直于 XY 平面」**指的是 **法向**（主要沿 **Z**），对应用户 **正对墙面、平视** 看内容时的视线方向——**不是**「从上往下俯视桌面」那种关系。

**产品上的相机约束：**

- 编辑体验以 **接近平视这面墙**（允许很小范围的斜视带来纵深），**不应偏离「沿 +Z 看向墙」这条法向太远**；理想实现是 **以该法向为轴的圆锥（锥体内可动）**，而不是在某种球坐标里对两个角各自 `min/max` 一切——后者和「绕视轴的锥体」几何上一般 **不等价**。
- **@react-three/drei** 的 Orbit 类控制器：**球坐标角** 已与 **`THREE.Spherical` 对齐**，本仓库的 **`theta`/`phi` 与其同名量一致**；但它仍偏 **对 polar/azimuth 做区间限制**，没有现成的「绕视轴 **+Z 锥体**」API，且整体交互与 **竖直工作面** 的心智仍不贴。锥体与物理整合在自研 **`tick`** 里做（见 [camera-architecture.md](./camera-architecture.md)）。

纵深、星野、地面等 3D 元素服务于 **氛围与可读性的平衡**，而不是替代 React Flow 的 2D 精度。

## 5. 美学基调：理科科幻 / Future's Theorist Sci-Fi

在 UI 材质与元素的表达上，PerSpatia 严格遵循 **“理科科幻” / Future's Theorist Sci-Fi** 取向，彻底摒弃“工科科幻”（Engineering Sci-Fi）的拟物遗留：

- **拒绝工科拟物**：UI 绝不是现实物理面板的投影。拒绝生硬的厚度、铆钉、机械感滑块、或是带有现实划痕与油污的材质。
- **拥抱绝对几何与能量流**：视觉构成应是极简的（甚至可以是简单的矩形等基础几何体），但质感必须是“非实体”的。例如发光的霓虹矢量线（Neon Vectors）、纯粹的光晕（Bloom）、无重量感的玻璃/力场、以及完全由物理和数学法则驱动的流畅动效。
- **核心心智隐喻（Cherry-picked Reference）**：视觉气质可部分参考刘慈欣《诗云》中神级文明“李白”出场时的场景——悬浮的完美发光球体与无限延伸的纯粹几何平面。这代表了科技发展到极致后，对物理与数学法则的绝对掌控与优雅（纯粹的几何与规则具象化）。**特此排雷**：虽然原著及《三体》水滴经常使用“绝对光滑的镜面”这一意象，但我们**明确拒绝镜面/反射材质**，因为它既不利于清晰地可视化数理概念，也会在 UI 中引入毫无意义的视觉噪点。我们只取其“绝对几何与能量实体”的骨相。
- **Least Cues 延伸**：视觉上的“指示词（Cues）”也应降至最低。不需要多余的边框去强调“这是一个容器”，让发光的边界或能量流转自然地界定空间。
- **实践与演进（Living Guideline）**：上述审美取向并非死板的教条，而是当前探索的基点（Baseline）。随着我们在实际代码和视觉原型（Variants）中的不断尝试，这些对光影、材质和几何的理解**预期将在实践中被持续挑战和更新**。务必将此视为一份会不断进化的活文档。
