# 移动端适配笔记 (Mobile Adaptation Notes)

为了让 PerSpatia 在移动端也能进行预览和基础交互，我们需要记录并处理一些移动端特有的适配问题。

## 1. 路由方案 (Routing)
- **HashRouter**：为了更好地适配 `live-server` 等静态服务器，已全面切换为 **`HashRouter`**。现在刷新页面时路径不会丢失。

## 2. 全局多指手势 (Global Multi-touch Gestures)
- **设计初衷**：为了让用户在移动端的操作更加**自由与自然**。在移动端，若不加处理，用户必须寻找“空白区域”才能进行缩放或旋转，否则手指一旦落在节点上就会变成拖拽节点，这极大地限制了操作自由度。我们需要修改交互机制，让用户在捏合或平移时，不需要刻意寻找“空白区域”以避开节点，而是可以在屏幕任何位置直接操作。
- **技术实现：Capture 阶段拦截**：
    - 我们在 `CameraControl` 中使用了 **`onPointerDownCapture`**。
    - **逻辑**：一旦屏幕上出现 >= 2 根手指，立刻拦截并停止事件下发。
    - **效果**：用户可以不用考虑节点的位置，直接在屏幕任意位置大方地进行捏合（缩放）或多指平移/旋转。节点在多指模式下会自动“退让”，从而实现无缝的相机控制体验。
- **防滑脱 (Pointer Capture)**：在拦截瞬间必须调用 `setPointerCapture`，确保手指划出视口后交互逻辑不中断。

## 3. OS 级手势冲突 (OS Gesture Conflicts)
- **核心痛点**：iOS 和部分安卓系统会拦截“原生三指下滑/捏合”等手势（用于截屏或系统快捷操作）。
- **终极解决方案**：**建议用户在手机设置中搜索“截屏”或“手势”，关闭所有与“三指”相关的系统快捷操作**。
- **备选技巧**：若无法关闭系统手势，可尝试“先放双指，再落第三指”的顺序操作，这通常能绕过 OS 的系统级手势识别。

## 4. 局限性与已知问题 (Known Issues)

### 4.1 节点边缘调节失效
- **现象**：在 PC 端可以通过拖拽节点边缘完美调整宽度，但在移动端触屏下该操作几乎无法工作。
- **分析**：推测为 React Flow 内部的 Touch 事件处理机制与自定义 Resize 逻辑存在深层冲突。
- **计划**：目前将其记录为已知局限。长远愿景中，若要彻底解决，可能需要自研 Node 层或改用 Flutter 渲染方案。

## 5. 开发工作流 (Workflow)
- **推荐方案**：运行 `vite build --watch` 并在另一个终端运行 `live-server` 托管 `dist` 文件夹。
- **优点**：手机加载快，且具备近乎实时的自动刷新体验。
