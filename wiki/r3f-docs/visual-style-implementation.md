# 视觉风格实现 (Visual Style Implementation)

本文记录 PerSpatia 视觉风格的具体技术实现细节，包括字体驱动与节点视觉呈现惯例。

---

## 1. 字体系统与字重控制 (Font System & Weight Control)

### 1.1 全局字体与字重清理 (Global Font & Weight Cleanup)
*   **核心痛点**：当前应用的字体族（`font-family`）和字重定义（`font-weight`）散落在各处，充斥着硬编码（Hard-coded）或不一致的声明。
*   **工程任务**：这是一项极其繁杂且琐碎的细节清理工作。必须走进应用的每一个角落，逐一巡检 CSS。
*   **视觉验证 (Visual Verification)**：单纯的代码巡检并不足够。必须通过**实际运行时的视觉检查和截图比对**，覆盖每一个功能模块和边缘场景，确保所有自建样式在视觉上都真正收口至底层变量，消除任何“漏网”的硬编码定义。

### 1.2 统一控制入口 (Unified Control Entry)
*   **愿景**：提供一个全局的字体与字重调节入口，允许系统或用户一键切换全应用的视觉密度与文字性格。
*   **前提条件**：完成 1.1 节的全局清理，确保全应用对变量的响应是同步、准确且受控的。


### 1.3 视觉字重归一化 (Visual Weight Normalization)
*   **必要性**：支持混合排版并保证视觉整洁。
*   **实现思路**：针对特定字体（如 Cascadia）引入偏移量（Offset）或映射系数，将全局的“抽象字重等级”转换为该字体特定的 CSS 数值。
*   **目标**：最终拿到一个带有简单操纵入口、能一键控制全应用视觉密度的应用形态。

### 1.4 兼容性与回退
*   **失效处理**：记录动态字重在特定浏览器环境、或非可变字体下的失效边界，提供静态阶梯作为降级方案。

---

## 2. 节点视觉呈现 (The Floating Node)

### 2.1 去实体化 (De-materialization)
*   **视觉逻辑**：节点根容器禁用 `background`, `border`, `box-shadow`。节点应像悬浮在空中的全息图。
*   **实现**：节点背景透明，不使用封闭边框，利用内部元素界定边界。

### 2.2 调节边界与装饰
*   **调节控件**：`NodeResizeControl` 保持透明且覆盖全高度。
*   **视觉引导**：使用发光的细线（如内部的 `node-decor-line`）来暗示结构，而非封闭的方框，增强“理科科幻”的悬浮感。

---

## 3. R3F 空间氛围场 (R3F Ambient Implementation)

### 3.1 调色盘与变体 (Color Variants)
*   **设计参照**：深度参考 **DATA WING** 的色彩美学，将空间视为具有情绪的容器。
*   **多配色支持**：背景不局限于单一色调。系统应具备切换不同氛围场（如 Warm Amber, Deep Cold, Nebula Pulse 等）的能力。
*   **技术路线**：背景由 R3F 环境组件驱动（而非简单的 CSS），利用 WebGL 的光效、体积感或自定义 Shader 来构建具有深度的空间景观。

### 3.2 视觉固化过程
*   **字重确定**：`--spatial-font-weight` 仅作为开发阶段的调参工具，用于在不同缩放层级下测试最佳视觉密度。
*   **固化原则**：一旦在特定配色方案中确定了最佳字重，应将其作为常量固化，而非在运行时动态改变。
